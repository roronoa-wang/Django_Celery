第三步

当用户收到短信验证码后，点击下一步，即对用户的身份进行核实，确定是否是用户本人。一旦确定是用户本人，便返回用于设置密码的接口调用凭据access_token。

后端接口设计：

请求方式： GET accounts/(?P<account>\w{5,20})/password/token/?sms_code=xxx

请求参数：

参数名 类型 是否必须 说明
account str 是 用户名或手机号
sms_code str 是 短信验证码

返回数据：

返回值 类型 说明
user_id int 用户编号
access_token str 发送短信的凭据

在users/serializers.py中定义序列化器

class CheckSMSCodeSerializer(serializers.Serializer):
    """
    检查sms code
    """
    sms_code = serializers.CharField(min_length=6, max_length=6)

    def validate_sms_code(self, value):
        account = self.context['view'].kwargs['account']
        # 获取user
        user = get_user_by_account(account)
        if user is None:
            raise serializers.ValidationError('用户不存在')

        redis_conn = get_redis_connection('verify_codes')
        real_sms_code = redis_conn.get('sms_%s' % user.mobile)
        if real_sms_code is None:
            raise serializers.ValidationError('无效的短信验证码')
        if value != real_sms_code.decode():
            raise serializers.ValidationError('短信验证码错误')
        return value

在模型类User中实现生成修改密码的token

class User(AbstractUser):
    ...
    def generate_set_password_token(self):
        """
        生成修改密码的token
        """
        serializer = TJWSSerializer(settings.SECRET_KEY, expires_in=constants.SET_PASSWORD_TOKEN_EXPIRES)
        data = {'user_id': self.id}
        token = serializer.dumps(data)
        return token.decode()

在users/views.py中定义视图

class PasswordTokenView(GenericAPIView):
    """
    用户帐号设置密码的token
    """
    serializer_class = serializers.CheckSMSCodeSerializer

    def get(self, request, account):
        """
        根据用户帐号获取修改密码的token
        """
        serializer = self.get_serializer(data=request.query_params)
        serializer.is_valid(raise_exception=True)

        user = serializer.user

        # 生成修改用户密码的access token
        access_token = user.generate_set_password_token()

        return Response({'user_id': user.id, 'access_token': access_token})
